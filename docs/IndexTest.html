The IndexTest program can be used to index a list of files or URLs or to recursively index a directory structure. Currently, we can index html and plain text documents.

Options

The IndexTest program takes a number of options:

Argument Description
-i <stuff> This option is required. The value of this option should be the name of a file containing a list of files. Each file in the list should appear on a separate line.
-d <dir> This option is required. The value of this option specifies the directory where the index will be placed. If the directory does not exist, it will be created. If the directory contains an existing index, that index will be added to.
-x <XML configuration file> Specifies an XML  file to be used as the basis for the properties given to the indexer. This may be an easier way to specify options. Details of the contents of this file are given in the Configuration Management for Sphinx-4 web page.
-e <entry type> This option specifies the type of entry that should be used in the main dictionary while indexing. The value of the entry is the name of the class to use for the entries. The default value is com.sun.labs.minion.indexer.entry.CasedDFOEntry.
-n <number> This option specifies the number of threads to use during indexing. Generally, using more threads will increase the indexing speed, although this appears to not be the case at the current time (locking problem?). The default value is 1.
-m <number> Specifies the amount of memory (in megabytes) to use to store index data before dumping a partition into the index. The default value is 64. Using a higher number will mean that you should provide more heap space to your JVM.
-k <directory> The directory where lock files will be placed.

Operation

As IndexTest is running, you will see progress messages printed out, indicating how much text has been indexed and how fast it has been indexed. You will also see messages about partitions that have been dumped into the index as well as merges of smaller partitions into larger ones.

You may run the QueryTest program at any time during the indexing run to query the index being built.
